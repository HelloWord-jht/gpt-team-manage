# Team Bus 全站视觉升级与续费工作台设计

## 目标

在不改变现有账号、成员、月份查询、成本换算和邮件提醒规则的前提下，完成一次全站视觉升级，并将右侧抽屉改为居中账号弹框与成员二级弹框。新增续费工作台，让待续费信息、提醒发送和当前周期处理状态可以集中管理。

## 范围

本轮包含：

- 全站采用“清爽运营台”视觉方向。
- 重构顶部栏、指标区、筛选区、状态标签、账号表格和空状态。
- 将账号新增/编辑抽屉改为居中主弹框。
- 将成员新增/编辑改为主弹框之上的二级弹框。
- 新增续费工作台，展示待续费车辆、成员、金额、提醒状态和处理状态。
- 支持手动发送当前待续费摘要。
- 支持将一辆车的当前续费周期标记为已处理。
- 保持现有每日扫描、提前三天提醒和每车每周期只自动发送一次的规则。

本轮不包含：

- 直接向成员发送邮件。
- 成员收款、退款或流水账。
- 登录和权限管理。
- 修改现有 Excel 导入数据模型。
- 将“标记已处理”自动解释为成员下车、账号停用或已收款。

## 视觉系统

整体采用高密度、安静、工作导向的管理台风格：

- 顶栏使用深色中性色，承载产品名称和主要操作。
- 页面主体使用浅灰背景与白色数据表面，不使用大面积渐变或装饰图形。
- 绿色用于主要操作和正常状态。
- 橙红色用于三天内续费和异常提醒，且同时使用图标、文字或边框，不能只靠颜色表达。
- 表格保持扫描效率，缩短无效留白，金额和日期使用稳定的数字排版。
- 圆角控制在 4 至 8 像素，按钮和输入框维持一致高度。
- 桌面端优先，同时保证 375 像素宽度下无文本重叠；窄屏表格允许横向滚动。

## 页面结构

### 顶部栏

- 左侧显示 Team Bus 管理台品牌和当前页面名称。
- 右侧保留刷新、新增账号和续费提醒入口。
- 续费提醒入口同时显示当前待处理数量。

### 指标区

保持四个紧凑指标：

- 活跃账号数。
- 当前月份在车成员数。
- 当前月份真实利润。
- 三天内待续费车辆数。

### 筛选与表格

- 搜索、月份、地区筛选放在同一操作带。
- 状态筛选作为紧凑分段标签，显示数量。
- 三天内待续费账号置顶，使用左侧警示边和浅色行底。
- 其他正常账号排在待续费账号之后，异常和停用账号排在最后。
- 行操作使用图标按钮，并提供可访问名称和悬浮提示。
- 下次续费列继续显示续费日期和倒计时。

## 弹框交互

### 账号主弹框

- 桌面端宽度约 820 至 920 像素，最大高度不超过视口。
- 顶部固定标题和关闭按钮，底部固定取消与保存按钮，中间内容可滚动。
- 账号邮箱、开通日期、状态、地区、成本、利润和备注在主弹框中编辑。
- 成员区只显示成员摘要列表，包括名称、邮箱、续费价格、上车和下车日期。
- 每个成员提供编辑和移除操作；新增成员按钮打开成员二级弹框。
- 账号保存前，成员修改仅存在于当前主弹框草稿中。

### 成员二级弹框

- 在账号主弹框上方打开，使用更深一级遮罩明确层级。
- 包含成员名称、邮箱、续费价格、上车日期和下车日期。
- 邮箱可为空，但填写后必须符合邮箱格式。
- 下车日期可为空，填写时不得早于上车日期。
- 确认后只更新主弹框草稿，不立即写入服务器。
- 取消后丢弃本次成员编辑，保留账号主弹框中的其他未保存内容。
- 按 Escape 只关闭当前最上层弹框。
- 打开时焦点进入二级弹框，关闭后回到触发按钮。

### 删除确认

账号删除不再使用浏览器原生确认框，改为小型确认弹框。确认文案明确显示账号邮箱，删除按钮使用危险操作样式。

## 续费工作台

续费工作台通过顶部“续费提醒”按钮打开居中大弹框。

### 待处理列表

每辆车按当前续费周期展示一行：

- 账号邮箱和地区。
- 下次续费日期和倒计时。
- 当前周期内的成员名称与邮箱。
- 每名成员续费价格及车辆合计价格。
- 自动提醒是否已发送及发送时间。
- 当前周期是否已处理。

默认只显示三天内且未处理的车辆，并按续费日期升序排列。工作台可切换查看本周期全部记录，包括已处理记录。

### 手动发送

- 手动发送继续给 `REMINDER_TO` 配置的车主邮箱发送摘要，不向成员发送。
- 邮件沿用当前专业提醒模板和发件人显示名。
- 若没有待发送车辆，按钮禁用并说明原因。
- 发送成功后立即刷新列表中的发送状态。
- 自动发送仍遵守每车每续费周期只发送一次。
- 手动发送默认也遵守去重规则，避免重复轰炸；工作台不提供强制重复发送入口。

### 标记已处理

- “已处理”只表示车主已完成当前车辆当前续费周期的人工核对。
- 标记后该车辆退出默认待处理列表，但保留在本周期全部记录中。
- 标记不会修改账号状态、成员上下车日期、续费价格或邮件发送历史。
- 下一续费周期会生成新的周期键，并重新进入待处理队列。
- 允许撤销当前周期的已处理状态。

## 数据与接口

### 周期键

继续使用 `accountId:nextRenewalAt` 作为车辆续费周期键。邮件发送记录和处理记录都以该键关联，互不覆盖。

### 本地存储

- 现有 `data/reminder-history.json` 继续保存邮件发送记录。
- 新增 `data/renewal-actions.json` 保存处理记录。
- Docker Compose 和 CentOS 7 启动方式继续挂载整个 `data/` 目录，因此记录可以跨容器重启保留。

### API

新增接口：

- `GET /api/renewals?month=YYYY-MM`：返回工作台记录、发送状态和处理状态。
- `POST /api/renewals/:cycleKey/handled`：标记当前周期已处理。
- `DELETE /api/renewals/:cycleKey/handled`：撤销已处理。

保留现有手动发送提醒接口。发送成功后前端重新请求续费列表，不在客户端猜测状态。

## 状态与错误处理

- 页面、账号弹框和续费工作台分别显示加载状态，避免整个页面被单一请求锁死。
- 表单校验错误显示在对应字段附近，保存失败时保留所有草稿。
- 网络或服务器错误显示可关闭通知，不清空当前弹框。
- 标记已处理和撤销操作在请求期间禁用对应按钮，避免重复提交。
- 邮件发送失败时保留待处理状态，并展示服务器返回的可操作错误。
- 数据文件不存在时按空记录启动；内容损坏时返回明确错误并写日志，不静默覆盖原文件。

## 可访问性与响应式

- 所有弹框使用 `role="dialog"`、`aria-modal="true"` 和明确标题关联。
- 弹框打开后锁定背景滚动并管理焦点。
- 图标按钮提供 `aria-label` 和 `title`。
- 键盘用户可以完成新增账号、编辑成员、发送提醒和标记已处理。
- 支持 `prefers-reduced-motion`，弹框动画控制在 150 至 220 毫秒。
- 375、768、1280 和 1440 像素宽度下验证无文字重叠和操作遮挡。

## 测试与验收

### 自动测试

- 领域测试覆盖周期键、待处理筛选、处理状态和跨周期重新出现。
- API 测试覆盖续费列表、标记、撤销、无效周期键和文件错误。
- 现有邮件去重测试保持通过。
- 前端逻辑测试覆盖成员草稿新增、编辑、取消和确认。

### 浏览器验收

- 新增和编辑账号不再出现右侧抽屉。
- 成员二级弹框关闭后，账号主弹框草稿仍然存在。
- Escape 只关闭最上层弹框。
- 三天内待续费车辆在首页置顶并在工作台默认可见。
- 标记已处理后退出默认列表，撤销后重新出现。
- 手动发送后发送时间更新，刷新页面后状态仍保留。
- 桌面和移动视口无文本重叠、按钮溢出或遮罩层级错误。

## 后续候选功能

以下功能保留到后续版本：

- 成员收款状态和欠款提醒。
- 退款、补差价和月度流水。
- 月度利润趋势与按地区成本分析。
- CSV 导入导出和自动备份。
- 汇率手动覆盖与历史汇率审计。
